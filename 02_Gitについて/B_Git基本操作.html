Gitの基本コマンド

branch · push · pull · merge — 4つのコマンドで始めるバージョン管理

🌿
branch — 分岐を作る
git branch / git checkout -b

ブランチは作業の「写し」です。mainを汚さずに新機能や修正を試せます。作業が終わったらmergeして本流に戻します。

$git branch

現在のブランチ一覧を表示。* が現在地

$git checkout -b feature/login

新しいブランチを作って、そこに移動

$git checkout main

既存のブランチに移動

⬆
push — リモートに送る
git push

ローカルのコミットをGitHub等のリモートリポジトリに送信します。チームメンバーが自分の作業を見られるようになります。

$git push origin main

mainブランチをリモートに送る

$git push origin feature/login

featureブランチをリモートに送る

$git push -u origin feature/login

-u で追跡設定。次回から git push だけでOK

⬇
pull — リモートから取得
git pull

リモートの最新変更をローカルに取り込みます。チームで作業するとき、他の人の変更を自分の環境に反映させるために使います。

$git pull

現在のブランチの最新を取得

$git pull origin main

リモートのmainを明示的に指定して取得

🔀
merge — ブランチを合流させる
git merge

featureブランチでの作業が完了したら、mainに取り込みます。まずmainに移動してから、mergeするのがポイントです。

$git checkout main

まずmainブランチに移動

$git merge feature/login

featureブランチをmainに取り込む

$git branch -d feature/login

マージ済みブランチを削除（任意）

1
リポジトリの初期化とファイル作成
mkdir git-practice && cd git-practice
git init
echo "はじめてのGit練習" > memo.txt
git add memo.txt
git commit -m "最初のコミット"

フォルダを作り、memo.txt を作成してmainブランチに最初のコミットをします。

2
branch 新しいブランチを作る
git checkout -b feature/add-line
echo "新しい行を追加しました" >> memo.txt
git add memo.txt
git commit -m "2行目を追加"

feature/add-line ブランチに切り替えて、memo.txt に1行追記してコミットします。

3
push リモートに送る（GitHubがある場合）
git remote add origin https://github.com/yourname/repo.git
git push -u origin feature/add-line

GitHubリポジトリがある場合はpushできます。ない場合はこの手順をスキップしてもOKです。

4
pull リモートの変更を取得
git checkout main
git pull origin main

mainブランチに戻り、リモートの最新を取り込みます。チーム開発ではmerge前にこれをやるのが習慣です。

5
merge mainに取り込む
git checkout main
git merge feature/add-line
cat memo.txt

memo.txt に2行目が追加されていれば成功！ブランチでの変更がmainに反映されました。

6
後片付け（任意）
git branch -d feature/add-line
git log --oneline

マージ済みのブランチを削除。git log でコミット履歴を確認してみましょう。
