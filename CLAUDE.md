# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## リポジトリの概要

これは新人研修向けのハッカソン資料リポジトリです。Vue.js 製 Todo アプリを改修する1日ハッカソンで使う、HTML 形式の教材・手順書を管理しています。コードそのものではなく、研修資料（HTML ファイル）が成果物です。

## セットアップ（参加者向け Todo アプリ）

参加者が取り組む Vue.js アプリは別リポジトリ（講師から案内される URL）にあります。そちらのセットアップ手順：

```bash
git clone <講師から指示があったリポジトリURL>
cd <クローンしてできたフォルダ名>
npm install
npm run dev
```

## このリポジトリの構成

```
index.html                    # 資料ポータル（全資料へのハブ）
00_事前準備/                  # Node.js・Git 環境構築手順
01_全体の流れ/                # タイムライン・ルール
02_Gitについて/               # Git 概念・コマンドリファレンス
03_課題セットアップ/
  A_今日のシナリオ.html       # 背景・依頼内容・AI 利用ルール
  B_プロンプトログの残し方.html
  C_Gitブランチのルール.html
  D_リポジトリ取得手順.html
  E_Todoアプリ機能仕様書/     # 課題 01〜10 の仕様書
04_クロスレビュー/
05_発表/
```

## Git 運用ルール

ブランチ構成：`main` → `develop` → `feature/*` / `docs/*`

- `main` への直接コミット禁止
- 作業は `develop` から feature ブランチを切って進める
- ブランチ名: `feature/task-delete`、`feature/task-check` など（`E_Todoアプリ機能仕様書/` の課題番号に対応）
- コミットメッセージ形式: `[機能名] 何をしたか`

## 課題の難易度区分

| レベル | 課題 |
|--------|------|
| ★1 必須 | 01 タスクの削除、02 完了チェック、03 優先度・色分け |
| ★2 追加 | 04 検索、05 完了非表示トグル、06 期限設定、07 複合ソート |
| ★3 エクストラ | 08 インライン編集、09 データ永続化、10 ドラッグ&ドロップ |

## 資料の編集方針

- 各資料は独立した HTML ファイル（CSS インライン、外部依存なし）
- デザインカラー: `#1a1a1a`（黒）/ `#fafaf8`（背景）/ `#ebebeb`（ボーダー）
- `index.html` は全資料へのハブ。新しい資料を追加した際はここにもリンクを追加する
