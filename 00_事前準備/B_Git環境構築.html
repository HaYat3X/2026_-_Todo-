Gitってなに？

ソースコードの変更履歴を管理するツールです。いつ・誰が・何を変えたかを記録し続けます。

GitHubはその記録をインターネット上に置いてチームで共有するサービスです。今日のハッカソンではチーム全員が同じリポジトリで作業するため、Git操作が必須です。

第1章　Gitのインストール

公式サイト https://git-scm.com/ にアクセスして「Download for Windows」からインストーラーをダウンロードします。

ウィザードは基本的にすべてデフォルトのまま「Next」で進めてください。以下の3か所だけ確認してください。

①
エディタ選択
初心者は「Notepad」を選択
②
初期ブランチ名
「Let Git decide」を選択
③
PATH設定
「Git from the command line and also from 3rd-party software」を選択

💡 インストール後はコマンドプロンプトまたはPowerShellを再起動してください。

インストールの確認：

git --version

git version 2.x.x と表示されれば成功です。

初期設定（名前とメールアドレスを登録）：

git config --global user.name "あなたの名前" git config --global user.email "your.email@example.com"

設定の確認：

git config --list

1

第3章　GitとGitHubの連携（SSH）

SSHキーを生成してGitHubに登録することで、パスワードなしで安全に通信できます。

1
SSHキーを生成する
コマンドプロンプトまたはPowerShellを開いて実行してください（メールアドレスはGitHubに登録したものを使用）。
ssh-keygen -t ed25519 -C "your.email@example.com"

ファイルの保存先を聞かれたらそのままEnter。パスフレーズはセキュリティのため設定を推奨しますが、Enterを2回でスキップも可能です。

2
公開鍵を表示してコピーする
type %USERPROFILE%\.ssh\id_ed25519.pub

ssh-ed25519 AAAA... で始まる文字列が表示されます。この文字列をすべてコピーしてください。

3
GitHubに公開鍵を登録する
GitHubにログイン → 右上アイコン → Settings → SSH and GPG keys → New SSH key

Title に分かりやすい名前（例：My Windows PC）を入力し、Key に先ほどコピーした公開鍵を貼り付けて「Add SSH key」をクリックします。

4
接続テストを実行する
ssh -T git@github.com

初回は「Are you sure you want to continue connecting?」と聞かれるので yes と入力してEnter。

Hi <ユーザー名>! You've successfully authenticated...

上記が表示されれば設定完了です。

💡 パスフレーズを設定した場合は、初回のみ入力が求められます。
