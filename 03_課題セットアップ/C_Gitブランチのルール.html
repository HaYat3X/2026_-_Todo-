ブランチの役割

ブランチ名 役割
main 本番リリース用。直接コミット禁止。
develop 統合・動作確認用。各 feature ブランチのマージ先。
feature/* 機能単位の開発ブランチ。develop から切る。
docs/* 機能単位のドキュメント更新用ブランチ。develop から切る。

feature ブランチ一覧

★1 基本機能
feature/task-delete
1. タスクの削除
feature/task-check
2. タスクの完了チェック
★2 応用機能
feature/task-priority
3. 優先度の設定と色分け表示
feature/task-filtering
4. タスクの検索（フィルタリング）
feature/task-toggle
5. 完了済み非表示トグル＋件数表示
feature/task-deadline
6. 期限設定と期限切れの警告表示
★3 発展機能
feature/task-sort
7. 優先度＋期限での複合ソート
feature/task-inline-edit
8. タスクのインライン編集
feature/task-persist
9. データの永続化
feature/task-dnd
10. タスクのドラッグ＆ドロップ並び替え
運用ルール

作業は必ず develop から feature ブランチを切って進める。main への直接コミットは禁止。マージ前にチームへ一声かけること。

ブランチ名・コミットメッセージはルールを守って記録を残してください。
