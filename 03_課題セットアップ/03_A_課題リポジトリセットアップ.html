Vue.js 環境構築手順書

Node.js が入っていない Windows 11 PC を前提に、Node.js のインストールから、GitHub 上の ToDo アプリをクローンし、 npm install と npm run dev で起動確認するところまでを順番に進める手順書です。

対象: 新人研修受講者 前提: Windows 11 構成: Node.js + npm + Vue 3 + Vite 想定所要時間: 15〜25分
概要 事前確認 手順 トラブル対応 参考リンク

この手順書で行うこと

この手順書では、公式の Node.js インストーラーで Node.js と npm を導入し、GitHub 上の ToDo アプリ vue-todo-app をクローンしたあとで npm install → npm run dev を実行して起動確認します。 Vue アプリの開発サーバーとして Vite が使われ、通常は http://localhost:5173 で起動します。

使用するもの

Node.js 24 LTS
npm は Node.js に含まれます。

使うプロジェクト

vue-todo-app
GitHub リポジトリをクローンして使用します。

確認すること

ブラウザで初期画面が見えること
npm run dev が通れば成功です。

事前確認

1. 使用する PC

  • Windows 11 が起動していること
  • インターネットへ接続できること
  • ブラウザが使えること
  • PowerShell が起動できること

2. 今回の前提

  • Node.js はまだ入っていない想定です
  • npm を別で入れる必要はありません
  • PowerShell は通常の起動で進めて問題ありません
  • 必要な場合だけ、インストーラー側で管理者権限の確認が表示されます

環境構築の手順

1

Node.js のダウンロードページを開く

Node.js の公式ダウンロードページを開き、LTS の版を使用します。 2026年4月時点では Node.js 24.14.1 が最新 LTS です。

Node.js 公式ダウンロードページを開く Node.js 24.14.1 (LTS) のリリース情報
今回ダウンロードするもの
研修で使う PC は共通で 64bit 版であるため、Windows Installer (.msi) / x64 を選んでください。
念のため確認する場合は、[設定] > [システム] > [バージョン情報] を開き、「システムの種類」 に 「64 ビット オペレーティング システム、x64 ベース プロセッサ」 と表示されているか確認してください。
2

Node.js をインストールする

  1. ダウンロードした .msi ファイルをダブルクリックします。
  2. 画面の案内に従って進めます。
  3. 基本的には既定値のままで問題ありません。
  4. インストールが完了したら、いったん PowerShell を閉じます。
  5. PowerShell を開き直します。
補足
通常は管理者権限で PowerShell を開く必要はありません。必要な場合は、インストーラーの途中で Windows の確認画面が表示されます。
3

Node.js と npm が入ったか確認する

PowerShell を開いて、次のコマンドを実行します。

node -v
npm -v

数字が表示されれば成功です。例: v24.14.1、11.x.x など。

4

GitHub から ToDo アプリをクローンする

次のリポジトリを任意の場所へクローンします。 ここでは例として、$HOME\Desktop に vue-todo-app をクローンします。

GitHub リポジトリを開く 
cd $HOME\Desktop
git clone https://github.com/HaYat3X/vue-todo-app
補足
クローンすると、通常は vue-todo-app というフォルダ名で作成されます。
5

クローンしたフォルダへ移動し、依存関係を入れる

cd .vue-todo-app
npm install
6

開発サーバーを起動する

npm run dev

開発サーバーが起動すると、通常は http://localhost:5173 が表示されます。 この URL をブラウザで開き、Vue の初期画面が表示されれば成功です。

成功の目安
PowerShell にローカル URL が表示され、ブラウザで http://localhost:5173 を開いて画面が見えれば環境構築は完了です。
7

ここまでのコマンドをまとめて確認する

node -v
npm -v
cd $HOME\Desktop
git clone https://github.com/HaYat3X/vue-todo-app
cd .vue-todo-app
npm install
npm run dev

トラブル対応

node または npm が見つからない

  • PowerShell を閉じて開き直す
  • Node.js のインストールが完了しているか確認する
  • それでも解消しなければ再インストールする

npm.ps1 cannot be loaded ... が出る

PowerShell の実行ポリシーが原因の可能性があります。

Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

会社 PC ではポリシーが管理されている場合があります。その場合は担当者へ確認してください。

5173 番ポートが使われている

たとえば次のような表示が出ることがあります。

Port 5173 is in use, trying another one...

  VITE v7.x.x  ready in xxx ms

  ➜  Local:   http://localhost:5174/

自分でポート番号を指定して起動したい場合は、次のように実行します。

npm run dev -- --port 5174

npm install に時間がかかる

  • 少し待っても進まないか確認する
  • 社内ネットワーク制限がないか確認する
  • 研修担当者へ状況を共有する
