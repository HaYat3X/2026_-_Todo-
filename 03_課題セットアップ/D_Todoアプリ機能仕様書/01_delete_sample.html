この資料の使い方

この資料は「完成コードを写すだけ」の資料ではありません。どう考えて、なぜその実装にしたかを体験してもらうために、AIとのやり取りと意思決定の流れをそのまま見せています。

仕様書を読み、チームで「何を作るか」の解釈をそろえる。

2

AIに方針を出させ、比較・検討して「なぜそうするか」をチームで決める。

3
docs に方針ログを書く → チューターに報告

決めた方針を docs/task-delete.md にまとめてから実装に入る。報告の条件はdocsが揃っていること。

4
実装する

サンプルコードを参考に、実際にコードを書く。コピペではなく手を動かして理解する。

5
docs に実装メモを書く → チューターに報告

実装して気づいたことを docs/task-delete.md に追記する。方針ログ＋実装メモが揃ったら最終報告。

STEP 1 — 仕様を確認する

まず仕様書の要件を読む。「何が求められているか」をチームて確認する。

要件（仕様書より）

・各タスクに対して削除できること

・削除したタスクは一覧から消えること

・ページのリロードは不要

チームで確認すること

削除ボタンを押したら即座に消えてよいか？「本当に削除しますか？」の確認ダイアログは必要か？
— 今回はシンプルにするため、確認ダイアログなし・即時削除で進める。

STEP 2 — AIと壁打ちして方針を決める

AIにいきなりコードを出させるのではなく、まず実装方針と複数案の比較を依頼する。以下はそのやり取りの記録。

AIへのプロンプト例

ToDoアプリを作成しています。

環境：Vue 3 / TypeScript / script setup

現状
- tasks という配列にタスクが格納されている
- v-for でリスト表示している
- 入力欄＋ボタンで配列にデータを追加している

今やりたいこと
- ユーザーが任意のタスクを削除できるようにしたい
- 各タスクごとに削除できること
- 削除したタスクは一覧から消えること
- ページのリロードは不要

お願いしたいこと
- 削除機能の実装方針を複数案で比較してほしい
- まだコードは出さず、方針整理と比較検討だけでよい
- 不明な点は「確認事項」として挙げてほしい

AIが提案する主な方針（要約）

案 方法 メリット デメリット
A インデックス指定 実装が最も簡単 並び替えやフィルタを追加するとズレが起きる
B 一意なID指定 確実に対象を特定できる。拡張性が高い 登録時にIDを付与する処理が増える
C オブジェクト参照 IDなしで特定できる indexOfで位置を探す手間がある
チームの意思決定

案B（一意なID指定）を採用する。
理由：今後フィルタリングやソートを追加したときにインデックスがズレる問題を避けたい。IDで管理すれば「どのタスクを消すか」が明確になり、コードの意図も読みやすい。実装の手間は少し増えるが許容できる。

ID生成方法もチームで決める

案Bを採用するには、タスクに一意なIDを付ける必要がある。AIに聞くと主に3案が出てくる。

方法 特徴
Date.now() 現在時刻のミリ秒。手入力での連打はほぼ起きないので実用上問題なし。シンプル。
crypto.randomUUID() ブラウザ標準のUUID生成。重複確率がほぼゼロ。モダンな書き方。
インクリメント連番 わかりやすいが、削除後の「次の番号」管理が煩雑になる。
チームの意思決定

Date.now() を採用する。
理由：手入力での追加なので同一ミリ秒での連打はほぼ起きない。コードが最もシンプルになる。今回の目的（削除機能の実装体験）に対して十分な精度がある。

STEP 4 — 実装する

方針が決まったら実装に入る。以下のサンプルコードは参考実装。コピペで終わらせず、コメントを読みながら「なぜこう書くか」を理解すること。

変更のポイント（全4箇所）

① Taskの型定義を追加　② tasks の型を変更　③ addTask にIDを追加　④ deleteTask を新規追加＋templateを修正

① Task型の定義を追加する

tasks の中身を「文字列」から「ID付きオブジェクト」に変える。まず型を定義する。

// タスク1件分のデータ構造
interface Task {
  id:   number;  // Date.now() をIDに使うので number 型
  text: string;  // 画面に表示するタスク名
}

② tasks の型を変更する

// 変更前: const tasks = ref<string[]>([]);
// 変更後: Task オブジェクトの配列に変更する
const tasks = ref<Task[]>([]);

③ addTask にID付与を追加する

const addTask = () => {
  if (newTask.value.trim() === "") return;

  tasks.value.push({
    id:   Date.now(),         // 追加：現在時刻をIDとして付与
    text: newTask.value.trim(), // 変更：文字列ではなくオブジェクトで追加
  });

  newTask.value = "";
};

④ deleteTask を追加する

filter は「条件に合うものだけを残す」メソッド。「対象IDと一致しないものだけを残す」＝「対象IDを削除する」になる。

const deleteTask = (targetId: number): void => {
  // targetId と一致しないタスクだけを残す = targetId を削除する
  tasks.value = tasks.value.filter((task: Task) => task.id !== targetId);
};

⑤ template を修正する

<!-- tasks が1件以上あるときだけリストを表示 -->
<ul v-if="tasks.length > 0">
  <li
    v-for="task in tasks"
    :key="task.id"
    class="task-item"
  >
    <!-- task はオブジェクトなので .text でテキストを取り出す -->
    <span>{{ task.text }}</span>

    <!-- 削除ボタンを押したタスクの id を deleteTask に渡す -->
    <button class="delete-button" @click="deleteTask(task.id)">削除</button>
  </li>
</ul>

<p v-else class="empty">タスクがまだありません。</p>
